### Substrate Pattern

2. **Identify the path** - What is the drill sequence?
3. **Check for patterns** - Is this a single point or regex match?

### While Coding:

2. **Verify coordinates** - Use `dimension.path` to confirm location

2. **Trace the path** - Log `dimension.path` at each step
3. **Check version lineage** - Is this the version you expect?
4. **Verify the coordinate** - Are you at the right point?

## 🏷️ MANIFOLD EXEMPTION DIRECTIVES

Not all code can or should use manifold patterns. Use these markers:

```typescript
// @manifold-exempt: [reason]              - General exemption
// @manifold-exempt-math: [reason]         - Pure math, no state access
// @manifold-exempt-external: [reason]     - External API integration
// @manifold-exempt-performance: [reason]  - Performance-critical path
```

### When to Use Exemptions:

| Directive | Use Case | Example |
|-----------|----------|---------|
| `@manifold-exempt-math` | Pure calculations | Vector math, matrix ops |
| `@manifold-exempt-external` | Third-party APIs | WebGL, Web Audio API |
| `@manifold-exempt-performance` | Hot paths | Render loops, audio buffers |

### Example:

```typescript
// @manifold-exempt-math: Pure vector operations, no manifold state
function dot(a: Vec3, b: Vec3): number {
  return a[0] * b[0] + a[1] * b[1] + a[2] * b[2];
}

// @manifold-exempt-external: WebGL API uses its own patterns
gl.bindBuffer(gl.ARRAY_BUFFER, buffer);
gl.drawArrays(gl.TRIANGLES, 0, 6);
```

### Test Compliance

Every folder should have manifold compliance tests:
- `tests/manifold-compliance.test.ts` - Core engine tests
- Tests verify: drill access, pattern matching, versioning
- Exempted code is documented but not tested for manifold patterns
