## 1. The Manifold (Root Reality)

The manifold is the base mathematical space defined by z = x \* y on a
7‑section helix. It is not storage. It is not a buffer. It is not a
serialization format.

The manifold is the _generative substrate_ from which all values are
discovered by following path expressions.

The manifold provides:
• dimensional structure
• coordinate addressing
• section ontology (Void → Whole)
• inflection boundaries
• geometric determinism

The manifold does NOT store data. It reveals data when traversed.

---

## 2. Primitive Substrates (Dimensional Building Blocks)

Primitive substrates are dimensional constructs built on top of the
manifold. Each substrate is a higher‑dimensional interpretation of the
previous one.

0D — Point substrate
A single coordinate on the manifold.

1D — Linear substrate
Ordered points forming a path (time series, animation curves,
audio waveforms, camera rails).

2D — Planal substrate
A grid or surface of linear substrates (images, heightmaps,
UI layouts, tilemaps).

3D — Volume substrate
A stack of planes forming a cubic region (voxels, physics
spaces, spatial audio fields, 3D worlds).

Whole/Object substrate
A 3D volume treated as a single point in the next dimension
(meshes, rigid bodies, characters, rooms, levels).

Dimensional rule:
• A substrate in dimension N is a point in dimension N+1.

---

## 3. Engines (Coordinated Substrate Systems)

Engines are not separate systems. They are orchestrators of substrates
moving through the manifold.

Examples:

• Physics engine:
Operates on volume + object substrates.
Forces, collisions, and integration are manifold paths.

• Music / sound engine:
Operates on linear substrates (time) and planal/volume substrates
for spatial audio. Frequencies are manifold paths.

• 3D substrate / renderer:
Operates on volume + object + camera substrates.
Rays, projections, shading are manifold paths.

• Camera engine:
A linear substrate over volume/object substrates.
Camera motion is a path expression.

• Game engine:
Coordinates all substrates and engines simultaneously.

All engines are dimensional. All engines use the manifold as the
underlying coordinate reality.

---

## 4. Representation Model (Not Compression)

The manifold is a representation model:
• It does not store bytes.
• It stores path expressions.
• A value is obtained by evaluating a path expression on the manifold.
• The representation may be smaller than raw data because the manifold
encodes the rules that generate the data, not the data itself.

This is analogous to physical laws:
• The universe does not store every particle.
• It stores the rules that generate them.

The substrate records _where_ in the manifold a value lives.

---

## 5. Path Expressions (Canonical Data Representation)

A datum is represented as a path expression:

section → which helix section
angle → rotation along the helix
radius → distance from center
depth → dimensional drill
modifiers → optional composition

Evaluating a path expression yields the value:
• number
• string
• color
• frequency
• object
• binary
• any semantic unit

The substrate stores path expressions, not payloads.

---

## 6. Reasoning Requirements for the AI

The AI must:
• Treat the manifold as the base reality.
• Treat substrates as dimensional constructs.
• Treat engines as orchestrators of substrates.
• Treat data as path expressions into the manifold.
• Never treat the manifold as a byte container.
• Never assume 1:1 storage with raw binary.
• Always reason dimensionally.

The AI must NOT:
• interpret the manifold as a serializer or codec.
• assume data is stored directly in memory.
• flatten dimensional structures into JSON-like forms.

---

## 7. Core Principle

The manifold is the universe.
Substrates are the dimensions.
Engines are the laws of motion.
Data is the path you take through it.
